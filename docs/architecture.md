# システムアーキテクチャ (System Architecture)

本システムは、C言語のクライアントアプリケーションとPostgreSQLサーバによる2層アーキテクチャ構成を採用しています。

## 1. レイヤー構成
* **Application Layer (C言語)**
  * ユーザーとの対話(CLI)
  * 入力値のサニタイズ（型チェック・バッファ管理）
  * ビジネスロジック（試験日までの残り日数計算のトリガー等）の実行
  * SQLクエリの動的生成
* **Interface Layer (libpq)**
  * C言語プログラムとPostgreSQL間の通信プロトコルを仲介
  * クエリの送信、および結果セット(`PGresult`)の受信・メモリ管理の制御
* **Database Layer (PostgreSQL)**
  * データの永続化
  * 参照整合性の維持 (`ON DELETE CASCADE`)
  * 高度なデータ処理 (集計処理、日付計算、外部結合)

## 2. データ構造とリレーション
* **certifications (資格マスタ)**: 資格名、目標日、現在の状態を保持。
* **study_logs (学習履歴)**: 各資格に紐付く学習時間とメモを保持（1対多のリレーション）。

## 3. データフロー
1. **Input**: ユーザーがCLIから操作（登録・表示等）を選択し、データを入力。
2. **Build**: C言語側でプレースホルダ(`$1`, `$2`)を用いたセキュアなSQL文を構成。
3. **Execute**: `libpq` を経由してPostgreSQLへクエリを送信し、エンジン側で実行。
4. **Aggregate**: DB側でマスタと履歴を結合・集計し、計算済みデータを出力。
5. **Output**: `PGresult` をC言語側でパースし、CLI上に整形して表示。
